# server/utils/totp.js

## File Overview

**File path:** `server/utils/totp.js`

Thin wrapper around the `otplib` library that provides all cryptographic operations needed for time-based one-time password (TOTP) two-factor authentication and for generating fallback recovery codes and email verification codes. The module has no state of its own; all functions are pure utilities.

**Dependencies (internal):** None.

**Dependencies (external):**
- `otplib` (`authenticator` object)
- `crypto` (Node.js built-in)

**Side effects when loaded:** None.

None at module level.

## Functions & Methods

### `generateSecret()`

**Signature:** `function generateSecret(): string`

**Returns:** A Base32-encoded TOTP secret string generated by `otplib`.

**Description:** Delegates directly to `authenticator.generateSecret()`. The returned value should be stored (encrypted) on the User document and shown to the user during 2FA setup.

**Callers:** `server/routes/auth.js` (TOTP setup endpoint).

---

### `generateTotpUri(secret, email, issuer?)`

**Signature:** `function generateTotpUri(secret: string, email: string, issuer?: string = 'StandUpTracker'): string`

**Returns:** An `otpauth://totp/...` URI suitable for embedding in a QR code.

**Description:** Calls `authenticator.keyuri(email, safeIssuer, secret)`. Sanitises the issuer string by replacing `:` and `/` characters with underscores to avoid malformed URIs.

**Callers:** `server/routes/auth.js` (TOTP setup endpoint, to return QR data to the frontend).

---

### `verifyTotp(token, secret)`

**Signature:** `function verifyTotp(token: string, secret: string): boolean`

**Returns:** `true` if the 6-digit token is valid for the current (or adjacent) time window, `false` otherwise.

**Description:** Delegates to `authenticator.verify({ token, secret })`. `otplib` applies a ±1 window tolerance by default.

**Callers:** `server/routes/auth.js` (TOTP login verification and enable verification).

---

### `generateRecoveryCodes(count?)`

**Signature:** `function generateRecoveryCodes(count?: number = 8): string[]`

**Returns:** Array of `count` uppercase hex recovery codes (8 hex characters each, e.g. `'A3F2C1D0'`).

**Description:** Each code is generated from 4 cryptographically random bytes. Codes are shown to the user once during 2FA enrollment and must be hashed before storage.

**Callers:** `server/routes/auth.js` (TOTP enable endpoint).

---

### `generateEmailCode()`

**Signature:** `function generateEmailCode(): string`

**Returns:** A 6-digit numeric string (e.g. `'483921'`).

**Description:** Uses `crypto.randomInt(100000, 1000000)` to generate a cryptographically random integer in the range [100000, 999999], which always produces exactly 6 digits.

**Callers:** `server/routes/auth.js` (email 2FA code send endpoint).

## Known Issues & Technical Debt

- The default `otplib` time window (±1 step = ±30 seconds) is not explicitly configured; if the operator needs a stricter or more permissive window there is no setting to change it.
- Recovery codes are returned in plaintext; the responsibility for hashing them before storage lies entirely with the caller in `auth.js`. There is no enforcement at this layer.
