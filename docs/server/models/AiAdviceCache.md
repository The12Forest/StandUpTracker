# server/models/AiAdviceCache.js

## File Overview

**File path:** `server/models/AiAdviceCache.js`

This file defines the Mongoose model for caching AI-generated advice per user. Each document stores one piece of advice for a user in a given context (e.g., `'dashboard'`). MongoDB's TTL index automatically deletes expired entries. The schema enforces a unique constraint per `(userId, context)` pair so there is at most one cached advice entry per context per user.

**Dependencies (external):**
- `mongoose`

**Side effects when loaded:** Registers the `AiAdviceCache` model with Mongoose and creates the two indexes on the collection.

## Classes & Models

### `AiAdviceCache`

**Collection name:** `aiadvicecaches`

| Property | Type | Required | Default | Description |
|---|---|---|---|---|
| `userId` | `String` | Yes | — | UUID of the user this advice is for |
| `context` | `String` | No | `'dashboard'` | The UI context the advice was generated for |
| `advice` | `String` | Yes | — | The generated advice text (Markdown) |
| `generatedAt` | `Date` | Yes | — | When the advice was generated by Ollama |
| `expiresAt` | `Date` | Yes | — | When the advice expires; used by both the app and the TTL index |
| `createdAt` | `Date` | Auto | — | Mongoose timestamp |
| `updatedAt` | `Date` | Auto | — | Mongoose timestamp |

**Indexes:**
| Index | Fields | Options | Purpose |
|---|---|---|---|
| Compound | `(userId, context)` | unique | Enforces one cache entry per user per context |
| TTL | `expiresAt` | expireAfterSeconds: 0 | MongoDB auto-deletes documents after `expiresAt` |

## Known Issues & Technical Debt

- The TTL index on `expiresAt` and the application-level cache check in `routes/ai.js` both enforce expiry independently. This is intentional redundancy but means the TTL cleanup is eventually consistent (MongoDB's TTL thread runs approximately every 60 seconds).
